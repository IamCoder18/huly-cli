The relevant SDK code lives in `node_modules/@hcengineering/api-client/lib/client.js`. Search for `processMarkup`:

```js
async processMarkup(_class, id, data) {
  const result = {};
  for (const [key, value] of Object.entries(data)) {
    if (value instanceof MarkupContent) {
      result[key] = this.markup.uploadMarkup(_class, id, key, value.content, value.kind);
    } else {
      result[key] = value;
    }
  }
  return result;
}

async createDoc(_class, space, attributes, id) {
  id ??= generateId();
  const data = await this.processMarkup(_class, id, attributes);

}
```

## Common pitfalls (apply these to the next session)

1. **Selfsigned MinIO bucket lifecycle is NOT in the Huly server** — set it up manually with `mc`.
2. **`WS_OPERATION=all` doesn't include `deletingSql` by default** (Fix #5 fixed this on our branch; `all+backup` was the workaround for original behavior).
3. **Cockroach `selfhost` user needs `CREATE` on `defaultdb`** — `grant` it once after `docker compose down -v`.
4. **The `redpanda` healthcheck using `rpk cluster info -X user=... -X pass=...` returns `ILLEGAL_SASL_STATE`** because of SASL bootstrap race conditions. Use unauthenticated metadata probe.
5. **Workspace pod version must match transactor version**, otherwise transactor's `sessionManager` rejects WebSocket connections. Bump `version.txt`.
6. **Nginx `proxy_pass <var>/;` (with trailing slash) replaces the matched location prefix with `/`** — so `GET /_transactor/api/v1/version` becomes `GET /` to the upstream. Use `proxy_pass <var>;` (no slash) and let the `rewrite` capture the path. Same applies to `location /` (root) — never use a trailing slash there.
7. **The SDK's `processMarkup` tries to upload MarkupContent to the collaborator service.** On this selfhost the collaborator throws. The CLI works around this by passing the body as a raw string instead of `new MarkupContent(...)`. If the collaborator gets fixed, you can revert to MarkupContent.
8. **CLI's `connectPlatform` needs both account token and workspace token cached.** Login gets account token; `selectWorkspace` gets workspace token. Both go in `~/.config/huly/credentials.json`. The `connectAccountCli` helper auto-prefers the workspace token if available, so account methods like `getWorkspaceMembers` work.
9. **`addCollection` returns a Ref<Doc>; `createDoc` returns a Ref<Doc>; `client.findOne` returns the doc or undefined; `findAll` returns an array.** Don't use `findAll` for "find one" — it's wasteful.
10. **The model is a 3-key stub on the CLI** (`getModel()` returns 3 keys) even though the transactor has 4080+ classes. The CLI uses `getHierarchy().hasClass(...)` for class lookups. `findAll` works because the transactor's live query layer is authoritative; the local model is just for the SDK's metadata validation.
